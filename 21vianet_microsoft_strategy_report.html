SECTION 02
出海算力通道：三层落地机制
1
双账户托管服务
世纪互联作为 Managed Service Provider，代理管理客户的 Azure Global 账户，提供中文支持与账单整合。无需修改现有协议即可落地。
现在可做
2
需求聚合 → 配额优先
聚合100+出海企业 GPU 需求，形成集团采购池，以规模承诺换取微软全球 H100/H200 优先配额，单企业无法单独谈判的条件集体实现。
现在可做
3
统一 OSPA 跨区条款
参考 Azure Stack 混合云 OSPA 模型，谈判将中国区与全球区算力消费合并在同一承诺合同下，客户一张合同、一个账单，世纪互联内部分配区域。
需与微软谈判
4
账户信用跨区继承
要求微软全球区认可中国区消费历史与信用评级，免除全球区 GPU 高级实例从零开始的资质审核等待期（目前可能长达数周）。
需与微软谈判
5
出海架构标准化路线图
针对游戏、电商、SaaS 出海等主流场景，预设"中国区 + 全球区"参考架构模板，标准化数据路由、身份认证、合规边界，大幅降低客户决策门槛。
现在可做
6
中国出海 AI 算力专区
在微软亚太节点（日本东/新加坡）争取"中国客户优先区"，世纪互联在该区域拥有销售代理权和客服权，实质将运营范围延伸至中国境外。
长期布局

SECTION 02
Microsoft Marketplace ISV 七大成功法则
①
可交易上架（Transactable）
不可交易的产品，微软销售不会有任何动力推广。Transactable 是解锁 Co-sell、MACC 抵扣、激励奖励的前提，没有例外。
核心前提
②
绑定 MACC 预算抵扣
企业客户已预承诺 Azure 消费额，ISV 产品若可抵扣该金额，采购等于"花掉已花的钱"，审批阻力极低，是最强的大客户促成机制。
成交加速器
③
激活 Co-sell 机制
微软销售卖方的配额与 ISV 的 Marketplace 交易额直接挂钩，获得 Co-sell 资质 = 免费获得一支全球销售军队，主动为你推客户。
销售杠杆
④
ISV + CSP 渠道组合
授权 Pax8、TD Synnex 等全球 CSP 转售，无需自建销售团队即可覆盖全球市场。LawToolBox 105% 增长、Nimble 进入 22 国均走此路径。
渠道杠杆
⑤
AI 产品获优先曝光
2025年9月统一 Marketplace 后，AI Apps & Agents 专区已有 3000+ 方案。与 Copilot 集成的产品可直接在 Microsoft 365 和 Azure 界面内嵌发现，流量优势显著。
AI 红利
⑥
最大化 ISV Success 激励
合格 ISV 可获高达 $128,000 资源包（Azure 信用 + 工具 + 咨询）；AI/Analytics 类产品额外可获 $10万 现金奖励；年交易额达 $4M 再加 $5万 返利。
资源红利
⑦
绩效导向：评分与留存驱动晋级
FY25起，只有产品评分 4.5+、有复购记录、有客户案例佐证的 ISV，才能进入最高激励层。参与是门槛，成果才是货币。
绩效导向
SECTION 03
中国 ISV 现状差距与世纪互联的杠杆
坦诚评估：目前中国 ISV 在微软全球 Marketplace 几乎没有可验证的成功 B2B 案例。微软中国区负责人亦坦承，微软在中国合作伙伴生态的短板正是缺乏具有全球化服务能力的合作伙伴——这既是差距，也是世纪互联的机会窗口。
障碍类型 具体问题 世纪互联的应对杠杆
账号体系隔离 中国区 Azure 账户无法直接在全球 Marketplace 上架，需重新注册全球区账户 双账户托管：以 MSP 身份代理管理客户全球区账户，提供中文支持和合规对接
产品英文化不足 UI、文档、技术支持、客户案例均缺乏英文版，直接影响 Marketplace 评分 英文化辅导计划：作为 ISV 上架加速服务，提供产品国际化改造支持
Co-sell 流程陌生 绝大多数中国 ISV 不了解 Co-sell 申请要求、Partner Center 操作流程 OCP 关系杠杆：利用世纪互联与微软 OCP 的深度关系，为中国 ISV 快速建立 Co-sell 资质
全球客户 Reference 缺失 进入 Marketplace 头部需要有客户案例，但中国 ISV 缺乏海外客户背书 首批客户孵化：以世纪互联自身或现有客户网络为中国 ISV 提供第一批海外试点客户
合规与法律壁垒 数据跨境、出口管制、GDPR 等合规要求对中国 ISV 而言过于复杂 合规框架包：预设不同市场（欧洲/东南亚/北美）的合规模板，大幅降低 ISV 进入门槛
